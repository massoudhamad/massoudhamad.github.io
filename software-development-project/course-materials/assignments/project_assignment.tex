\section{Project Overview}

\subsection{Introduction}
This project assignment requires students to work in teams to develop a complete software application following industry-standard software development practices. The project spans the entire semester and covers all phases of the Software Development Life Cycle (SDLC).

\subsection{Objectives}
By completing this project, students will:
\begin{itemize}
    \item Apply software engineering principles in a real-world context
    \item Practice Agile/Scrum methodology with sprint-based development
    \item Gain hands-on experience with version control (Git/GitHub)
    \item Develop teamwork and collaboration skills
    \item Create professional documentation
    \item Build a portfolio-ready software project
\end{itemize}

\subsection{Team Formation}
\begin{itemize}
    \item Team size: \textbf{3 members}
    \item Each team must assign roles: Product Owner, Scrum Master/Developer, Developer/QA
    \item Roles can rotate during the project
    \item All team members must contribute to coding
\end{itemize}

\section{Project Requirements}

\subsection{Project Scope}
Teams must develop a \textbf{web-based application} that solves a real-world problem. Suggested project domains include:

\begin{itemize}
    \item \textbf{E-commerce:} Online store, marketplace, booking system
    \item \textbf{Education:} Learning management, quiz platform, library system
    \item \textbf{Healthcare:} Appointment booking, patient records, pharmacy
    \item \textbf{Finance:} Expense tracker, budgeting app, mobile money integration
    \item \textbf{Agriculture:} Farm management, market prices, crop advisory
    \item \textbf{Tourism:} Hotel booking, tour packages, travel guide
    \item \textbf{Community:} Event management, volunteer coordination, local services
\end{itemize}

\subsection{Technical Requirements}

\subsubsection{Minimum Features (Must Have)}
\begin{enumerate}
    \item User authentication (registration, login, logout)
    \item User roles (at least 2: e.g., admin and regular user)
    \item CRUD operations for main entities
    \item Search and filter functionality
    \item Responsive design (mobile-friendly)
    \item Form validation (client and server-side)
    \item Database integration
\end{enumerate}

\subsubsection{Technology Stack}
Teams may choose from the following options:

\textbf{Frontend:}
\begin{itemize}
    \item HTML5, CSS3, JavaScript
    \item CSS Framework: Bootstrap, Tailwind CSS, or similar
    \item Optional: React, Vue.js, or vanilla JavaScript
\end{itemize}

\textbf{Backend:}
\begin{itemize}
    \item Node.js with Express.js
    \item Python with Django or Flask
    \item PHP with Laravel
    \item Java with Spring Boot
\end{itemize}

\textbf{Database:}
\begin{itemize}
    \item MySQL, PostgreSQL, or SQLite (relational)
    \item MongoDB (NoSQL - if appropriate)
\end{itemize}

\textbf{Version Control:}
\begin{itemize}
    \item Git and GitHub (mandatory)
    \item GitHub Projects or Trello for task management
\end{itemize}

\section{Submission Guidelines}

\subsection{Document Format}
\begin{itemize}
    \item All documents must be submitted as PDF
    \item Use the provided templates where available
    \item Include team name and member names on all documents
    \item Follow academic writing standards
\end{itemize}

\subsection{Code Submission}
\begin{itemize}
    \item All code must be on GitHub
    \item Repository must have a clear README.md
    \item Use meaningful commit messages
    \item Create branches for features (not all work on main)
    \item No sensitive data (passwords, API keys) in repository
\end{itemize}

\subsection{Late Submission Policy}
\begin{itemize}
    \item 10\% penalty per day for late submissions
    \item Maximum 3 days late accepted
    \item No submissions accepted after 3 days without prior approval
\end{itemize}

\section{Checklist for Students}

\subsection{Before Starting}
\begin{itemize}
    \item[$\square$] Form a team of 3 members
    \item[$\square$] Choose a project idea
    \item[$\square$] Create GitHub accounts (all members)
    \item[$\square$] Set up communication channels (WhatsApp, Discord, etc.)
\end{itemize}

\subsection{During Development}
\begin{itemize}
    \item[$\square$] Commit code regularly (at least daily during sprints)
    \item[$\square$] Attend all team meetings
    \item[$\square$] Update project board with task progress
    \item[$\square$] Document issues and decisions
    \item[$\square$] Test features before marking as complete
\end{itemize}

\subsection{Before Each Submission}
\begin{itemize}
    \item[$\square$] Review all requirements for the deliverable
    \item[$\square$] Proofread all documents
    \item[$\square$] Ensure code compiles and runs
    \item[$\square$] Get all team members to review
    \item[$\square$] Submit before deadline
\end{itemize}
